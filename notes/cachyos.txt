These commands fixed the issue:

    ###### Apply DOCKER-USER Rules
    sudo iptables -I DOCKER-USER -s 172.17.0.0/16 ! -o docker0 -j ACCEPT
    sudo iptables -I DOCKER-USER -d 172.17.0.0/16 ! -i docker0 -j ACCEPT

    ###### Enable Forwarding Persistently
    sudo sysctl -w net.ipv4.ip_forward=1
    echo 'net.ipv4.ip_forward=1' | sudo tee /etc/sysctl.d/99-docker.conf
    sudo sysctl --system

Immediately started working

    The DOCKER-USER Rules need to be ran on systems like cachyos

    The Forwarding could be used as a sysctl in docker-run or docker-compose instead

After realizing the fix wasn't persisting and troubleshooting, it ended up being rp_filter, which is set to 2
  by default in cachyos.

    sudo sysctl -w net.ipv4.conf.docker0.rp_filter=0
    sudo sysctl -w net.ipv4.conf.enp5s0.rp_filter=0  # Your NIC
    sudo sysctl -w net.ipv4.conf.all.rp_filter=0     # Catch-all for veth/wg0

    I put these in /etc/sysctl.d/99-docker.conf to make them persist

Turns out this was all BS
  All that was necessary to get it to start working was issuing:
    sudo sysctl --system

  After adding a systemd drop-in override for docker.service at
    /etc/systemd/system/docker.service.d/after-sysctl.conf
      Everything is working perfectly. The file simply runs the sysctl --system command after docker starts

  after-sysctl.conf:
    [Service]
    ExecStartPost=/usr/bin/sysctl --system
